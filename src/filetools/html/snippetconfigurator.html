Creat Map File

Interface for creating a map file.

Imapfile

Opens an Imap file that will be used to help complete other configuration fields such as species&tree and phase.

Imap file:

seed

Specifies the seed (a positive integer) used to initialize the random number generator. A negative integer argument specifies that the computer generate a random seed. A positive integer argument specifies a particular seed to use.



usedata

Use likelihood+prior for inference or instead turn off likelihood and use only prior (useful for debugging or checking prior distributions).

ON: Using Likelihood (Data) + Prior

OFF: Using Only Prior (No Data)

outfile

Specifies the name (and optionally a path) for a file that will receive output produced by the BPP program.

New output file:

mcmcfile

Specifies the name (and optionally a path) for a file that will receive the MCMC variable values (one line per iteration of the MCMC) produced by the BPP program.

New MCMC file:

seqfile

Specifies the name (and optionally a path) for a file that contains DNA or protein sequence data for input to the BPP program.

New sequence data file:

burnin

Specifies the number of MCMC iterations that should be discarded (as burn-in) before parameter sampling from the chain begins. Increase the burnin if the chain does not seem to be converging to a stationary distribution before sampling begins.

sampfreq

Specifies the intervals at which samples of parameters are collected from the MCMC and printed to the output file. Sampling at larger intervals reduces the autocorrelation of the samples and also the output file size.

nsample

Specifies the number of samples from the MCMC that are to be written to the output file (mcmcfile). The total number of MCMC iterations is burnin + nsample × sampfreq.

print

Specifies which variables are saved to the output file during an MCMC analysis.

ON: Saving MCMC samples (taus, thetas, trees)

OFF: Not saving MCMC samples

ON: Saving locus rates

OFF: Not saving locus rates

ON: Saving locus heredity scalars

OFF: Not saving locus heredity scalars

ON: Saving locus gene trees

OFF: Not saving locus gene trees

ON: Saving locus substitution parameters

OFF: Not saving locus substitution parameters

finetune

Specifies whether to automatically adjust step sizes for parameter proposals in MCMC or instead use manually specified step sizes.

ON: Automatically Adjust Step Sizes

OFF: Use Manually Specified Step Sizes

speciesdelimitation

Specifies whether to delimit species (ON) or to instead treat the assignments of sequences to species as known (OFF), and as specified in the Imap file.

ON: Delimiting Species

OFF: Treating Species as Fixed (Delimited)

speciestree

Specifies whether to infer species (ON) or to instead treat the species tree as fixed (OFF), and as specified in the control file.

ON: Inferring Species Tree

OFF: Treating Species Tree as Fixed

cleandata

Specifies whether columns in the alignment which have gaps or ambiguity characters will be removed prior to analysis, or will instead be retained for use in the likelihood calculation.

ON: Gaps/Ambiguity Sites Removed

OFF: Gaps/Ambiguity Sites Retained

nloci

Specifies the number of loci to be analyzed (must be less than the number of loci contained in the sequence data file). If nloci is less than the total number of loci in the sequence data file the first nloci in that file will be analyzed.

locusrate

Specifies the model of substitution rate variation among loci as well as the parameters and prior if required. There are 3 models: constant rates among loci, variable (estimated) rates among loci, and variable (specified) rates among loci. The variable specified rates option requires a file be specified in which each line contains a relative rate and the number of lines equals the number of loci.

If inheritance scalars are estimated, the parameters of the prior also must be specified.

File name:

species&tree

Description of species names, maximum number of sequences per species, and the species tree topology. The species tree is either the starting tree (for analyses A10, A11), the fixed tree for inferring demographic parameters (analysis A00), or the guide tree for delimitation (analysis A01). This snippet is automatically completed when a Imap file is specified using the REQUIRED (ND)::Imapfile menu item.

No Imap file. Configuring the Imapfile control file variable will autocomplete the species&tree control file variable.

phase

Specifies whether sequence data are phased or unphased for each species. The default is phased. This snippet is automatically made available for specifying the phase of individual species when a Imap file is specified using the REQUIRED (ND)::Imapfile menu item.

No Imap file (default: all species are phased). Configuring the Imapfile control file variable will make the options for specifying the phase of each species available.

tauprior

Specify the parameters alpha and beta of the gamma prior distribution for tau (age of root of species tree) in units of expected substitutions per site.

:Alpha
:Beta

alphaprior

Specify the parameters alpha and beta of the gamma prior distribution for the shape parameter alpha and the number of rate categories to use when modeling among-site rate variation using discrete gamma distribution.

:Alpha
:Beta
:No Categories (Discrete Gamma Approximation)
